HEDEF
Aşağıdaki sorunları çöz:

1. Server Component içinde next/dynamic({ ssr: false }) kullanımı var.
   Bu kullanım Next.js 16.1.6 altında build kırıyor.
   Özellikle app/ route’larında ve app/page.tsx benzeri dosyalarda bunu tespit et.
   Çözüm:
   - Server Component içinde ssr:false dynamic kullanımını kaldır.
   - Gerekiyorsa ilgili parçayı ayrı bir "use client" component’e taşı.
   - App router kurallarına tam uy.

2. Hydration mismatch var.
   Server ve client aynı HTML’i üretmiyor.
   Çözüm:
   - Tarih, random, window kontrolü, theme state, mounted state, localStorage, media query, browser-only logic kaynaklı farkları tara.
   - Browser-only davranışları client boundary içine al.
   - Gerekli yerlerde mounted guard kullan ama tüm sayfayı boş gösteren kötü çözümlerden kaçın.
   - Server/client render çıktısını stabilize et.

3. Global stil zinciri kopmuş.
   app/layout.tsx içinde globals.css importu ve body class yapısı bozulmuş olabilir.
   Çözüm:
   - app/layout.tsx dosyasını kontrol et.
   - globals.css importunu garanti altına al.
   - body/html className yapısını eski branded yapıya en yakın, tutarlı ve production-safe hale getir.
   - Font, antialiasing, background, text rendering zincirini düzelt.

4. Provider katmanı eksik.
   theme-provider, toaster ve root seviyede olması gereken provider’lar layout’a düzgün sarılmamış olabilir.
   Çözüm:
   - Root layout’ta hangi provider’lar olması gerekiyorsa tespit et.
   - Theme provider, toaster ve gerekli global UI provider’larını doğru sırada bağla.
   - Client/provider sınırlarını doğru kur.

5. Login ve bazı sayfalar branded görünümünü kaybetmiş.
   Çözüm:
   - Login ekranını ve ana giriş akışını incele.
   - Var olan eski branded componentler / style tokenları / ortak ui yapıları varsa onları yeniden bağla.
   - Minimal sade form görünümüne düşen yerleri branded hissi geri gelecek şekilde toparla.
   - Ancak yepyeni tasarım yazma; mevcut yapıyı restore et.

6. Production build tamamlanmıyor.
   pnpm build veya next build “Creating an optimized production build ...” aşamasında takılıyor.
   Çözüm:
   - Build’i kilitleyen exact sebebi bul.
   - Problemli import, recursive dependency, server/client boundary hatası, invalid dynamic usage, package uyumsuzluğu, bad effect, sonsuz render, config sorunu gibi nedenleri sistematik kontrol et.
   - Sorunun kaynağını net tespit etmeden rastgele dosya değiştirme.
   - Build geçene kadar devam et.

7. next start için geçerli production çıktısı oluşmuyor.
   Çözüm:
   - Geçerli .next production çıktısı oluştur.
   - next start komutunun gerçekten production çıktısı ile açıldığını doğrula.
   - 502 veya restart loop sebebi olabilecek eksikleri kapat.

8. PM2 yanlış moda düşmüş.
   Production yerine next dev ile geçici açılmış olabilir.
   Çözüm:
   - PM2 konfigürasyonunu incele.
   - Gerçek production çalıştırma komutunu kullan.
   - next dev kullanımını kaldır.
   - Süreç ismi, çalışma dizini, env, restart mantığını production-safe hale getir.

9. Renk, tipografi ve tasarım tokenları branded histen uzaklaşmış.
   Çözüm:
   - globals.css, tailwind/theme tokenları, css variables, radius, palette, typography ayarlarını incele.
   - Mevcut projede eski branded görünüme ait izler varsa onları restore et.
   - Nötrleşmiş tokenları eski marka hissine daha yakın hale getir.
   - Ama tüm sistemi yeniden tasarlama; sadece regressions düzelt.

10. No stdout uygulama hatası değil.
   Bu madde kod değişikliği istemiyor.
   Bunu yalnızca not olarak kabul et, Next.js runtime problemi gibi ele alma.
